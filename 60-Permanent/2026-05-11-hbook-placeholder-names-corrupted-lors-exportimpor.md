**Symptôme détecté** lors d'export Instance C → Instance D: les noms d'hébergement (accommodation names) affichent `{***HEX-VALUE***}` au lieu du nom réel dans HBook widget.

**Contexte**: Mutation de la mu-plugin `jt-hbook-redesign.php` entre les instances ou escaping/sanitization double-encoding les placeholders lors de l'import.

**Localisation du bug**: Probablement dans:
- Fonction export/import HBook
- Sanitization hook dans jt-hbook-redesign
- Différence de configuration entre Instance A (baseline) et Instance C/D

**À investiguer avant cutover**:
- Comparer fonction d'export en Instance A (correcke) vs Instance C
- Checker si placeholders sont échappés/double-encoded
- Tester avec données réelles de réservations

**Critère de succès avant cutover prod**: Noms d'hébergement visibles et corrects après import sur l'instance principale.

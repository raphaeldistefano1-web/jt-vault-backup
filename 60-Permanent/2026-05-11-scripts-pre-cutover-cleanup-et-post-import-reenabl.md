Création de scripts stockés `/var/www/jt-migrate-scripts/`:

**1. Pre-cutover cleanup** (`pre-cutover-cleanup.sql` + `pre-cutover-cleanup.sh`):
- Suppression des données obsolètes (sessions, drafts, logs temporaires)
- Optimisation tables (REPAIR, OPTIMIZE)
- Vérification intégrité DB
- Run avant l'export pour réduire footprint et risques

**2. Post-import reenable** (`post-import-reenable-emails.sh`):
- Réactivation des hooks emails après import
- Force resync user roles/capabilities
- Vérification notifications mail fonctionnent

**Avantage**: Automatise les étapes critiques et souvent oubliées, réduit erreurs manuelles, documenté pour futures migrations, reproductible.

**Reusability**: Modifiable selon instance source/cible mais structure générique.

Pattern observé 6 fois consécutives dans cette session: après validation d'une étape (implémentation, install plugin, etc.), user approuve puis **demande immédiatement** une comparaison de perf et une vérification fonctionnelle avant la prochaine étape.

Étapes types:
1. Implémentation/fix appliqué → approuvé
2. Test perf (comparison ancien/nouveau)
3. Vérification fonctionnelle (éléments critiques chargés, noms corrects, emails OK)
4. Approuvé → étape suivante

Ce workflow réduit le risque de regression et détecte les bugs avant cutover prod. User l'applique même pour les changements qu'il considère "mineurs".

**Critère de succès**: Aucune étape avancée sans test perf + vérification complète documentée.

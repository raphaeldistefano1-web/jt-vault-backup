Quand tu as un projet/plugin sans historique git existant et que tu veux utiliser ultrareview pour reviewer les changements accumulés, initialiser le repo:

```bash
cd /var/www/jt-migrate
git init -b main
git config user.email "raphael.distefano1@gmail.com"
git config user.name "Raphael Distefano"
git commit --allow-empty -m "init: empty baseline for ultrareview diff"
```

Le commit empty sert de baseline. Ultrareview génère alors un diff contre ce point de départ, révélant tous les changements du projet.

**Cas d'usage**: Avant major version release, ou quand reviewer un plugin avec accum de modifications sans git history.

**Rationale**: Ultrareview nécessite un repo git pour fonctionner; créer un baseline explicit permet un diff propre et complet.
